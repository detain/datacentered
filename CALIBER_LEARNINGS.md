# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha]** `start.php` loads TWO composer autoloaders: `__DIR__.'/vendor/autoload.php'` then `__DIR__.'/../../my/vendor/autoload.php'` (the shared MyAdmin vendor at `/home/my/vendor/`). When diagnosing class-loading failures, check BOTH `vendor/composer/autoload_static.php` and `autoload_classmap.php` files — a stale classmap in either one can point to paths that no longer exist.
- **[gotcha]** Workerman 5.x relocated source under a `src/` subdirectory (e.g. `workerman/workerman/src/Protocols/Http/Request.php`), while Workerman 4.x had `Protocols/Http/Request.php` directly under the package root. If `include()` warnings reference the old non-`src/` path, a classmap was generated against v4 and is now stale — run `composer dump-autoload -o` in the directory whose autoloader is stale (check both `/home/sites/datacentered/` and `/home/my/` if needed).
- **[pattern]** When the `include_path` shown in a PHP warning points at `/home/my/vendor/old_pear/...`, that means `/home/my/vendor/autoload.php` has already been loaded — so any class-resolution fault is coming from the chained MyAdmin autoloader, not just the local project's.
- **[gotcha]** The long-running `WorkerMan: master process` forks all workers at startup and keeps its Composer autoload map in memory. If `composer dump-autoload` is run on `/home/sites/datacentered/vendor/` or `/home/my/vendor/` AFTER the daemon is already running, existing workers will still resolve classes against the OLD classmap (producing `Failed to open stream` warnings for paths that no longer exist on disk). Fix: `php start.php restart` (graceful) after any composer update that touches a package used by the daemon.
- **[pattern]** To confirm a stale-autoload-in-running-process situation: compare the master PID's lstart with the mtimes of `/home/my/vendor/composer/autoload_static.php` and `/home/sites/datacentered/vendor/composer/autoload_static.php` — `ps -p <pid> -o pid,etime,lstart,cmd` vs `stat -c '%y %n' <autoload files>`. If the autoload file is newer than the process start, the process is running with a pre-update autoload map.
